# Material 3 Expressive Demo

This is a lightweight example note for the converter flow.

## Why it exists

- Lets you try a markdown to document route right away.
- Gives the preview panel something friendly to render.
- Helps show ETA and route reasons inside the new Tune step.

## Tiny checklist

1. Load the example.
2. Confirm the detected source.
3. Pick the suggested target.
4. Tune CPU usage.
5. Review the planned route.
